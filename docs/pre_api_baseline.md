## 5. What Changes After API Is Active

| Layer | Before | After |
|-------|--------|-------|
| **Verdicts** | Sentences assembled from counts: "X roles + Y launches = sustained R&D investment" | Reasoned narrative from actual signal combination — what this company specifically is doing and why it matters to Actian now |
| **Hiring briefs** | Same 6 template implications for every company | Specific inferences from actual job title clusters, seniority patterns, geo spread |
| **`watch_for`** | Generic: "track posting velocity", "watch for exec hires" | Observable leading indicators specific to this company's inferred strategy |
| **`recommended_actions`** | Generic: "refresh battlecard", "brief SDRs" | Non-generic: specific action, specific reason tied to this company's direction |
| **Event summaries** | Raw truncated page text (Korean characters, duplication) | One sharp sentence: what this event signals, which Actian team should care |
| **Event vs launch split** | 63 events : 9 launches — over-classifies events | LLM reads title + description together, separates recurring webinars from strategic events |
| **News summaries** | No summary — just a type label | One sentence per item explaining what happened and Actian relevance |
| **News misclassification** | Milvus open-source release → "pricing" | Correct type from context |
| **Actian relevance** | Binary keyword match — "data engineering" = high | Reasoned per item: does this specifically threaten Actian's catalog/observability/vector/integration product? |
| **Signal deduplication** | Pinecone event appears 5×, Alation 6×, Databricks 9× | LLM won't fix scraping dedup (separate issue), but summaries will be unique per item |
