WHAT'S NEW - 7 FIXES IMPLEMENTED
================================================================================

FIX 1: clean_title() Rewrite
├─ Strips Workday location codes (IND-CHENNAI • Hybrid)
├─ Strips MongoDB metadata (location Dublin suitcase)
├─ Strips generic bullets, parentheticals, trailing locations
└─ 5 comprehensive regex patterns

FIX 2: extract_location_from_title() NEW FUNCTION
├─ Extracts location BEFORE title cleaning
├─ Workday: IND-CHENNAI → "Chennai, India"
├─ MongoDB: "location Dublin suitcase" → "Dublin"
└─ Generic: "Title - New York, NY" → "New York, NY"

FIX 3: ATS JSON API Handlers - CRITICAL
├─ extract_lever_jobs() - Direct API for jobs.lever.co
├─ extract_greenhouse_jobs() - Direct API for boards.greenhouse.io
├─ API dispatch in scrape_all() with HTML fallback
└─ Full error handling + rate limiting

FIX 4: FORBIDDEN_TEXT_RE Non-Job Content Filter
├─ Blocks: demo, webinar, podcast, blog, whitepaper, ebook, etc.
├─ Applied in 3 places: link detection, title validation (2 paths)
└─ Reduces noise in output

FIX 5: Updated PLAYWRIGHT_DOMAINS
├─ Added: ashbyhq, onetrust, atlassian, microsoft, google
├─ New: PLAYWRIGHT_URL_DOMAINS for flexible domain matching
├─ Updated needs_playwright() to check both company + URL
└─ Better JS-heavy site detection

FIX 6: Workday-Specific Title Extraction
├─ Try Workday selectors: [data-automation-id="jobPostingTitle"], .css-1q2dra3, etc.
├─ Falls back to <h1> if selectors don't match
└─ For all myworkdayjobs.com URLs

FIX 7: Location Extraction from Title (Fallback Chain)
├─ Extract from detail page HTML (first priority)
├─ Fall back to extracted from raw title
├─ Fall back to location from job card context
└─ Default to empty string
